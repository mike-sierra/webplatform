Simple box dimensions, rounded corners, and shadows. Try adding a blur to a zero-offset shadow. Also, applying a blur to an inset shadow simulates the appearance of a gradient. Tap the example to add or remove a background image, to see how transparent shadow colors appear.

Tap the example to see how text effects appear against a background image. Text strokes affect the interior of each letterform, while text shadows only affect the exterior. The alternative text fill color functions the same as the better-known color property.

Tap the example to see how transparencies appear against a background image. The opacity (alpha) of a background color can be manipulated independently from the opacity of the overall element. HSL and HSLA reference a color's hue in degrees, saturation (vibrancy) and luminence (brightness) as percentages.

Gradients specify origin points, starting color, and destination color. Radial gradients specify the circle's size along with the origin points, which should be placed within the circle. Optional color stops allow for shifts to additional colors along the specified path.

Masks allow you to use data from one image to modify another. Reflections allow you to create mirror effects.

All background properties accept comma-separated arguments to specify multiple images. The cover and contain size options maintain the image's aspect ratio, while setting both x and y to 100% snaps the image to the size of the targeted element. The origin specifies what portion of the box against which to size the image: the inner content, or any additional padding or border area. The clip property hides image content outside the specified area. The composite property allows you to modify the image based on the background color.

Border images are used to fill border regions, based on a 3 × 3 grid applied to the image. Specify the image source, then how much of the image to slice off each side and place within the border region. The slice's fill keyword specifies whether to display the center portion of the grid. The repeat property specifies how to fill the sides (and filled center) once the corners have been sliced off. The width specifies the dimensions within the element's border box in which to extend the image, expressed as a multiple of the current border width. The outset likewise pushes the image outside the element's border box. The slice, width, and outset properties each accept up to four values to customize each side. The repeat property can accept two separate values for horizontal and vertical edges.

An outline can be styled much like a border, but doesn't affect the element's placement.

The text-overflow property allows ellipses to be inserted when text overflows the right margin, when white-space is set to no-wrap. The word-wrap property prevents long words from running past a column's width. To enable scrolling marquee animations, specify -webkit-marquee as any overflow property, and use marquee-specific properties to control their presentation. The repetition sets how many times the animation executes. The increment controls the distance jumoed in each frame.

When you change the number of its columns, the block may resize to accommodate the new dimensions. To demonstrate how various column break and span options affect the second heading, set the number of columns to 3 so that it appears at the bottom of a column by default, or 2 so that it appears within a column.

When tapped, the example animates from one set of CSS style sheets to another. Choose a specific property to animate, or all of them. Transition functions specify the animation's progress as a curve or as a sequence of steps. While this example specifies a single transition, each property accepts multiple, comma-separated values. Specifying delays for some transitions allows you to build sequences.

Animations are based on keyframe rules that define potentially complex animation sequences. Setting the direction to alternate means even-numbered iterations play back in reverse. The fill mode controls whether properties specified at the beginning of end of a keyframe sequence appear as defined before or after the animation executes. It applies only if there is a delay specified or if the iteration count is limited. The play state allows you to pause animations while in progress. Animations rely on the same set of timing functions as transitions. Some properties may not apply dynamically while an animation is running, which occurs when you change the animation's name.

Transforms specify different functions that displace or distort an element's appearance. The translate() function moves an element on the screen; scale() resizes it; skew() modifies its default 90° corners; rotate() spins it. Modify the transform origin if you want to, for example, rotate the element around a point other than the center of the element.

Similar to 2D transforms, but rotations, translations, and transform origin can extend along the z axis, towards or away from view.

A wrapper element requires perspective that specifies the number of pixels away from the viewer any transformed child elements appear. (Alternately, each transformed element can specify its own perspective.)

The main element's transform style specifies whether any transformed children render within its own 3D space, rather than flat against the element. With preserve-3d enabled, any of the nested child's transforms appear within the space defined by its parent's transforms.

The backface of any element that rotates away from view can be hidden from view, enabling flip-panel effects.

Flexible boxes offer layout options that are not available to standard block elements. Boxes can be oriented horizontally or vertically, controlling how child elements flow within the available space. The pack property allows you to distribute them evenly, and align allows you to cluster or stretch them along the perpendicular axis. The direction property allows you to reverse their order. The ordinal property allows elements to be ordered arbitrarily, grouped independently of their actual sequence in the DOM. The box-flex property increases their flexibility relative to other flexible elements. (Elements such as #1 and #4 whose box-flex is zero are inflexible.)

Specify whether elements can be dragged, of if their text can be edited or selected. For elements such as form inputs that present a custom interface, you can override their appearance and supply your own. Use the text-size-adjust property to keep text from increasing in size when zooming content, also useful when shifting to landscape orientation on mobile browsers.
